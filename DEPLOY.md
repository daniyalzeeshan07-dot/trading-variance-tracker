# Deploy to Vercel in 5 Minutes

Follow these exact steps:

## Step 1: Create GitHub Account (2 min)
1. Go to https://github.com
2. Click "Sign up"
3. Create account with your email
4. Verify email

## Step 2: Create GitHub Repository (1 min)
1. Go to https://github.com/new
2. Repository name: `trading-variance-tracker`
3. Description: "Trading variance tracker app"
4. Choose "Public"
5. Click "Create repository"

## Step 3: Upload Files to GitHub (1 min)
After creating the repo, GitHub shows you instructions. Instead:

1. Click the "uploading an existing file" link
2. Select ALL these files to upload:
   - package.json
   - next.config.js
   - tailwind.config.js
   - postcss.config.js
   - .gitignore
   - README.md
   - DEPLOY.md
   - pages/_app.js
   - pages/_document.js
   - pages/index.js
   - styles/globals.css

3. Click "Commit changes"

## Step 4: Deploy to Vercel (1 min)
1. Go to https://vercel.com
2. Click "Sign Up"
3. Click "Continue with GitHub"
4. Allow permissions
5. Click "New Project"
6. Select `trading-variance-tracker` repository
7. Click "Import"
8. Click "Deploy"

## DONE! ✅

Your app is now live! You'll get a URL like:
```
https://trading-variance-tracker-xxx.vercel.app
```

Save this URL. That's your app!

## Test It Out
1. Visit your Vercel URL
2. Add a test trade
3. Verify data saves
4. Share the link with anyone!

## Optional: Custom Domain
Once deployed, you can:
1. Buy a domain (Namecheap, Google Domains, etc)
2. Add it in Vercel project settings
3. Follow Vercel's DNS instructions
